1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "docmost.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status of by running 'kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "docmost.fullname" . }}'
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "docmost.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "{{ include "docmost.selectorLabels" . }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:8080 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 8080:$CONTAINER_PORT
{{- end }}

2. Important configuration notes:
   - Make sure to configure your database connection in values.yaml (secrets.databaseUrl)
   - Configure Redis connection in values.yaml (secrets.redisUrl)
   - Set a secure APP_SECRET (minimum 32 characters) in values.yaml (secrets.appSecret)
   - Update the APP_URL to match your domain in values.yaml (config.appUrl)
   
3. For production use:
   - Enable TLS/SSL on your ingress
   - Use external secrets management (e.g., External Secrets Operator, Sealed Secrets)
   - Configure proper resource limits
   - Set up proper monitoring and logging
   
4. Helpful commands:
   # Get pod details
   kubectl get pods -l "{{ include "docmost.selectorLabels" . }}" -n {{ .Release.Namespace }}
   
   # View logs
   kubectl logs -l "{{ include "docmost.selectorLabels" . }}" -n {{ .Release.Namespace }}
   
   # Describe deployment
   kubectl describe deployment {{ include "docmost.fullname" . }} -n {{ .Release.Namespace }}
